A ideia aqui e começar a construção de um projeto que vai evoluir com o passar do tempo conforme vou adquirindo/aprimorando novos conhecimentos. O objetivo principal e montar uma estrutura “profissional” no sentido de organização de entregas e comprimento de metas sejam elas referentes a código em si ou designer.

A principio o projeto será inteiramente focado em desenvolvimento front-end(implementações fullstacks para o futuro). A divisão inicial vai ser basear no comprimento de 4 etapas primordiais para o entendimento completo de cada funcionalidade adicionada ao projeto, uma vez que o foco do projeto e exclusivamente aprimorar e adquirir novos conhecimento.

As 4 etapas serão divididas da seguinte maneira: 

### Etapa 1

Pensar.

Aqui o principal e entender o que exatamente vai ser feito em relação ao contexto proposto ao projeto. Ou seja, qual é o principal objetivo que o projeto espera alcançar? e quais são as entidades que vão pertencer a esse objetivo para esse ser completado?
```bash
**qual é o principal objetivo que o projeto espera alcançar?**
```
> Criar um pequeno RPG em JavaScript para praticar programação e evoluí-lo conforme novos conceitos forem sendo estudados.
> 
```bash
**quais são as entidades que vão pertencer a esse objetivo?**
```
> Quais objetos (entidades) você acredita que existirão no nosso jogo e qual a responsabilidade de cada um?
> 

- Personagens - Um jogo de RPG não pode existir sem personagens, eles criam a narrativa da história, eles constroem o mundo ao seu redor, dentro desse projeto os personagens são a base para o funcionamento de todas as outra entidades que de forma singular vão de alguma forma interagir diretamente com esses personagens.

- Inimigos - Para a construção de enredo/narrativa você precisa construir situações onde se torna interessante acompanhar para onde aquela história vai. Inimigos são a base pra isso acontecer, atreveis deles você pode desenvolver mecânicas únicas tanto para aprimorar a experiencia de quem está jogando quanto implementar ganchos narrativos para a construção da sua historia.

Para um sistema simples de combate essas duas entidades são os pilares para de fato algo acontecer, porém ainda pode-se acrescentar algumas outras entidades para deixar a construção um tanto mais complexa e ao mesmo tempo ainda bastante simplificada como: 
```text
- Item
- Inventário
- classes
- raças
- diferentes sistemas de combate
- recompensas por vitorias
- etc…
```

### Etapa 2

Planejar.

O planejamento começa a parti do momento em que você entender qual e o objetivo daquilo que você esta construindo juntamente da descrição das entidades que vão fazer aquilo que você esta construindo se tornar possível.

No caso desse sistema de RPG simples em Java Script o principal na hora de planejar como ele será feito e primeiro entender quais são as limitações obvias que você vai encontrar. E um fato que no mundo real projetos são feitos utilizando infinitas tecnologias, infinitas ferramentas tanto para diversificar a construção da estrutura como um todo, mas principalmente para abranger um range maior de possibilidades levando em quanto o tamanho do escopo do projeto. Utilização de apenas uma ferramenta ao longo do desenvolvimento traz consigo um limitante no escopo gigantesco, que pode parecer algo ruim a primeira vista, mas que de certa forma ajuda a trabalhar a raciocínio na hora de resolver o problemas que certamente aparecerão.

A escolha de utilização apenas de ferramentas simples tem como objetivo alcançar 2 metas: 1. Aprimorar minha base de conhecimento nessas ferramentas que tem sua certa importância no mercado; 2. Não exagerar no tamanho do escopo, a meta e fazer algo cabível ao meu atual nível de conhecimento.
